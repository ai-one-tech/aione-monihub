**目标**

* 时间转为浏览器时区显示

* 页面新增 `application_name`、`instance.host_name`、`log_type` 列

* 内容、UA、IP、上下文列默认 150 宽度，超出隐藏，增加查看图标，点击弹出 Dialog；若为 JSON 用专业组件展示

**涉及页面与文件**

* 系统日志页与表格：`apps/frontend/src/features/system/logs/index.tsx`、`components/system-logs-table.tsx`

* 请求日志页与表格：`apps/frontend/src/features/system/logs/request-logs.tsx`、`components/request-logs-table.tsx`

* 通用 UI：`apps/frontend/src/components/ui/dialog.tsx`（弹窗）

**实现方案**

* 时间时区转换

  * 在两个表格组件中新增小工具函数，容错解析后端时间字符串（示例：`2025-11-17 01:54:50.522 +00:00`），标准化为 ISO，再用 `new Date(...).toLocaleString()` 格式化为本地时区。

  * 替换时间列渲染：

    * 系统日志：`system-logs-table.tsx:47` 将 `row.original.timestamp` 改为本地时区格式化结果。

    * 请求日志：`request-logs-table.tsx:40` 同上。

* 新增列

  * `log_type`：此页固定类型（系统/请求），在表格列中添加常量列，显示对应值。

  * `application_name`：利用现有 `useApplicationsQuery` 构建 `id -> name` 映射（`system-logs-table.tsx:87-93` 已有数据），在列渲染时用 `row.original.user_id` 映射名称，缺失显示 `-`。

  * `instance.host_name`：前端当前响应未包含 `instance_id`，先添加列占位显示 `-`；后续可在后端响应补充 `instance_id` 后用 `useInstancesQuery` 做映射显示主机名。

* 150 宽度、溢出隐藏、Dialog 查看

  * 为列渲染引入一个轻量的“溢出预览”式渲染：内容容器 `w-[150px] truncate`，尾部放一个 `Eye` 图标按钮；点击打开 `Dialog`（`components/ui/dialog.tsx`）。

  * Dialog 中：

    * 若值为对象或可 `JSON.parse`，用第三方 JSON 组件展示；否则使用 `<pre>` 文本展示。

  * 替换/调整列：

    * 系统日志：`action`（内容）、`user_agent`（UA）、`ip_address`（IP）列的宽度改为 `w-[150px]` 并使用溢出预览 + Dialog。

    * 请求日志：`path`、`user_agent`、`ip_address` 改为 `w-[150px]`；`request_headers`、`request_body`、`response_body` 也用溢出预览 + Dialog，并在 Dialog 内使用 JSON 组件完整展示。

* JSON 专业组件

  * 引入 `react-json-view-lite`（体积小、易用）用于 Dialog 展示 JSON，非 JSON 使用 `<pre>`。

**兼容性与注意事项**

* 时间解析容错：对 `Chrono` 默认字符串做正则标准化，避免 Safari 对非 ISO 的解析失败。

* 现有筛选、分页不变；`application_name`、`instance.host_name` 仅影响展示，不影响查询参数。

* `instance.host_name` 与 `log_type` 更精准展示可在后端补齐字段（不影响本次前端改造）。

**验证**

* 打开系统/请求日志页：

  * 时间为本地时区、格式正确。

  * 新增三列出现且数据合理（无数据时显示 `-`）。

  * UA/IP/内容/上下文列宽度 150，超出隐藏，查看图标可弹窗；JSON 内容在 Dialog 中结构化显示。
