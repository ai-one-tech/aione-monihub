## 列表页（ConfigsTable）
- 列：`code`、`name`、`environment`、`config_type`、`version`、`updated_at`、`description`、操作列
- 工具栏：搜索（`code/name` 模糊）、按 `environment/config_type` 过滤、列显示切换
- 分页与 URL 同步：复用 `useTableUrlState`
- 时间格式化：`formatDateTime`（`@/lib/datetime`）

## 表单（ConfigsEditSheet）
- 字段：
  - 基本：`code`、`environment`（Select）、`name`、`config_type`（Select，仅 `object/array/html/text`）、`description`
  - 内容：`content`（
    - `object/array`：JSON 编辑（`Textarea` + JSON 校验），
    - `html/text`：`Textarea`
  )
  - 结构：`schema`（仅 `object/array` 显示，JSON 校验）
  - 选项值：`generate_values`（仅当 `config_type = array` 显示 Checkbox）
- 模式：`create/edit/view` 三种模式，`view` 只读；`edit` 拉取详情预填
- 提交：
  - 创建：`POST /api/configs`
  - 更新：`PUT /api/configs/{id}`（生成新版本，不覆盖旧记录）
- 校验：
  - 前端根据 `config_type` 做基本校验：
    - `object`：`content` 必须可解析为对象
    - `array`：`content` 必须可解析为数组；若 `generate_values` 勾选，数组项必须为对象且包含 `code`、`name`
    - `html/text`：不做结构校验
    - `schema`：如填写则必须是合法 JSON

## Hooks 与 API（统一使用 apiClient）
- 列表：`GET /api/configs`，支持查询参数：`page`、`limit`、`search`、`config_type`、`environment`、`all_versions`
- 详情：
  - 按 ID：若需要可添加 `GET /api/configs/{id}`（当前后端未提供，使用 code+env+version/最新）
  - 按 `code + environment` 最新版：`GET /api/configs/code/{code}/environment/{environment}`
  - 按 `code + environment + version`：`GET /api/configs/code/{code}/environment/{environment}/version/{version}`
- 创建：`POST /api/configs`
- 更新（新版本）：`PUT /api/configs/{id}`（接受请求体中的新内容）
- 删除：`DELETE /api/configs/{id}`（软删）
- 错误反馈：统一 `toast`；`react-query` 的 `onError` 做提示

## 交互细节
- 仅在 `config_type = array` 时显示并启用“生成选项值”开关；其他类型不显示该开关
- `schema` 字段仅在 `object/array` 显示；提交时如果为空则忽略
- 列表默认显示最新版本（`all_versions=false`），可切换查看所有版本（列出历史）
- 新建/更新成功后自动刷新列表并关闭抽屉；在 `view` 模式支持一键跳转到编辑
